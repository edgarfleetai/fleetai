# FleetAI Cloud Persistent DB FIX

Исправление проблемы: записи пропадали после обновления/деплоя сайта на Render, потому что SQLite база лежала внутри временной файловой системы проекта.

Теперь приложение автоматически использует постоянный путь:

```text
/var/data/fleetai.db
```

если на Render подключен Persistent Disk с mount path `/var/data`.

## Что сделать на Render

1. Откройте ваш сервис FleetAI на Render.
2. Перейдите в раздел **Disks**.
3. Нажмите **Add Disk**.
4. Укажите:

```text
Name: fleetai-data
Mount path: /var/data
Size: 1 GB
```

5. Сохраните.
6. Залейте этот проект на GitHub.
7. Сделайте Deploy.

После этого записи не будут пропадать при обновлении сайта.

## Локальный запуск

Локально база будет храниться в файле:

```text
fleet.db
```

Запуск:

```bash
python3 app.py
```

## Важно

Если раньше записи уже пропали, восстановить их можно только из старой базы, если она где-то сохранилась.
После подключения Persistent Disk новые записи будут сохраняться постоянно.
